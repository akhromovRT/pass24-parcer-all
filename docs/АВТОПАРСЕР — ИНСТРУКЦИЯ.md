# Автопарсер КП МО — Инструкция

**Что это:** Система, которая автоматически 2 раза в неделю находит новые коттеджные посёлки и ТСН/СНТ Московской области и создаёт лиды в Bitrix24 на стадии **«Парсер»**, ответственный — **Павел Мельников**.

---

## Часть 1 — Настройка (один раз, для Алексея)

### Шаг 1. Открыть настройки репозитория

Перейти по ссылке:
```
https://github.com/akhromovRT/pass24-parcer-all/settings/secrets/actions
```

Нажать **New repository secret** для каждого из 4 секретов:

| Имя секрета | Значение |
|---|---|
| `BITRIX24_WEBHOOK_URL` | `https://pass24pro.bitrix24.ru/rest/1/k8s07s2shaitmhmy/` |
| `BITRIX24_RESPONSIBLE` | `16` |
| `DADATA_API_KEY` | *(из файла `.env` в проекте)* |
| `DADATA_SECRET_KEY` | *(из файла `.env` в проекте)* |

> Поле **Value** — вводить точно, без пробелов. После сохранения значение больше не видно.

### Шаг 2. Запустить первый тест вручную

Перейти:
```
https://github.com/akhromovRT/pass24-parcer-all/actions/workflows/parse-leads.yml
```

Нажать **Run workflow** → выбрать режим `dadata-only` → **Run workflow**.

Ждать 5–15 минут. Зайти в Bitrix24 → Лиды → фильтр **Стадия = Парсер** → убедиться, что появились новые лиды.

## Часть 2 — Для Павла Мельникова

### Как это работает

Каждый **понедельник и четверг в 04:00** система автоматически:

1. Запрашивает базу ЕГРЮЛ через dadata.ru — находит ТСН, СНТ, ДНП Московской области
2. Для каждой организации берёт: **ФИО председателя**, ИНН, адрес, название
3. Проверяет, нет ли уже такого в Bitrix24 (по телефону, email, ИНН)
4. Проверяет, не является ли это уже нашим клиентом (по названию сделок)
5. Создаёт **лид в Bitrix24**: стадия «Парсер», ответственный — ты

---

### Где смотреть новые лиды

**Bitrix24 → CRM → Лиды → Канбан**

Фильтр: Стадия = **Парсер** (колонка сразу после «Новый»)

Каждый новый лид содержит в **комментарии**:
- Регион (Московская область / Москва)
- Источник данных (ЕГРЮЛ/dadata или Сайт КП)
- ФИО руководителя (председатель правления ТСН/СНТ)
- ИНН организации
- Адрес объекта
- Ссылку на ЕГРЮЛ для проверки: `https://egrul.nalog.ru/?query=<ИНН>`

---

### Что делать с лидом

**Задача:** позвонить председателю ТСН/СНТ или найти его телефон.

Порядок работы:

1. **Открыть лид** → прочитать комментарий: ФИО, ИНН, адрес
2. **Найти телефон** (если его нет в лиде):
   - Ввести ИНН на сайте `egrul.nalog.ru` → перейти на сайт организации
   - Или поискать название КП + «телефон» в Яндексе
   - Или позвонить через справочник 2ГИС по адресу
3. **Позвонить** в течение 24 часов (Speed-to-Lead)
4. **Квалифицировать** по CHAMP:
   - Challenges: есть ли проблема с контролем доступа?
   - Authority: это председатель или нужно выйти на него?
   - Money: есть бюджет / кто подписывает договор?
   - Priority: когда планируют решение?
5. **Перевести стадию** лида:
   - Квалифицирован → «Взят в работу»
   - Не актуально → «Не интересно» (заполнить причину)
   - Нет контакта → оставить в «Парсер», вернуться через 2 недели

---

### Что написать при первом контакте

> «Добрый день, [Имя]! Меня зовут Павел, компания PASS24 — мы делаем облачную систему контроля доступа для коттеджных посёлков. Видим, что вы управляете [название ТСН], хотел узнать — есть ли сейчас задача по системе въезда/выезда или домофонии?»

### Запустить вручную (если нужно срочно)

```
https://github.com/akhromovRT/pass24-parcer-all/actions/workflows/parse-leads.yml
→ Run workflow → выбрать режим → Run workflow
```

Режимы:
- `all-sources` — все источники (рекомендуется)
- `dadata-only` — только ЕГРЮЛ (быстро, ~10 минут)
- `seed-only` — только список сайтов КП

---

### Если что-то пошло не так

| Симптом | Что делать |
|---|---|
| Нет новых лидов после запуска | Открыть Actions → последний запуск → вкладка «Run parser» → посмотреть лог ошибок |
| Лид — уже наш клиент | Закрыть как «Не интересно — существующий клиент» |
| Дублирующийся лид | Это баг, написать Алексею |
| Нужно добавить новые сайты КП в список | Написать Алексею, он добавит в `data/seed_urls.txt` |
