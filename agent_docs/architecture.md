# Архитектура

## Обзор

PASS24 Parser — система сбора, обогащения и экспорта контактных данных ЛПР (лиц, принимающих решения) для продукта PASS24.online. Парсер работает в связке с двумя системами-потребителями: Bitrix24 CRM и AI Sales Factory.

```
Источники данных → [Сбор] → [Нормализация] → [Обогащение] → [Дедупликация] → [Квалификация] → Экспорт
                                                                                                  ├─→ Bitrix24 (CSV / REST API)
                                                                                                  └─→ AI Sales Factory (webhook JSON)
```

## Контекст

### Место в экосистеме PASS24

```
pass24-knowlege-base    — база знаний о продукте, ЛПР, конкурентах, методологии продаж
pass24-parcer-all       — ЭТОТ ПРОЕКТ: парсинг контактов ЛПР из открытых источников
pass24-ai-sales         — 12 AI-агентов автоматизации продаж (Lead Generator, Lead Scorer, Nurturing, Outreach и др.)
pass24pro.bitrix24.ru   — CRM, воронки продаж, управление сделками
```

### Целевые сегменты (по приоритету)

| Приоритет | Сегмент | ЛПР | Признаки целевого объекта |
|-----------|---------|-----|--------------------------|
| 1 (основной) | Коттеджные поселки (КП) | Председатель правления ТСН/СНТ, представитель УК | Построен и заселён, есть КПП/охрана, >30 домов |
| 2 | Жилые комплексы (ЖК) | УК или ТСЖ | Построен, есть консьержи/шлагбаумы |
| 3 | Бизнес-центры (БЦ) | УК или собственник здания | Действующий, с системой пропусков |

## Ключевые компоненты

### 1. Коллекторы (Collectors)
Модули сбора сырых данных из конкретных источников. Каждый коллектор — отдельный модуль с единым интерфейсом.

**Источники данных (по приоритету):**
- **Seed-список URL** (`data/seed_urls.txt`) — проверенные URL сайтов КП, прямой скрапинг. Основной источник для MVP (72 URL → 26 контактов)
- **DuckDuckGo поиск** — ICP-запросы по шоссе/районам МО, автоматическое обнаружение новых КП (2-6 КП за прогон)
- **2GIS API / парсинг** — адреса объектов, контакты УК (требует Playwright — JS-рендеринг)
- **Яндекс Карты** — геоданные, рейтинги, отзывы, контакты организаций (Фаза 2)
- **Сайты КП/ЖК** — страницы "Контакты", "Правление", "Руководство" — извлечение ФИО ЛПР
- **ЕГРЮЛ (nalog.ru / egrul.nalog.ru)** — юридические данные УК/ТСН, ФИО руководителей, ИНН/ОГРН
- **Социальные сети и форумы** — группы жителей КП в VK/Telegram (будущее)

### 2. Нормализатор (Normalizer)
Приведение данных из разных источников к единой внутренней модели:
- Стандартизация телефонов (→ формат +7XXXXXXXXXX)
- Стандартизация email (lowercase, trim)
- Стандартизация адресов (регион, город, объект)
- Определение типа объекта (КП/ЖК/БЦ)

### 3. Обогатитель (Enricher)
Дополнение данных из дополнительных источников:
- Поиск email по домену организации
- Поиск ФИО руководителя по ИНН/ОГРН через ЕГРЮЛ
- Определение размера объекта (кол-во домов/квартир)
- Определение наличия существующей СКУД

### 4. Дедупликатор (Deduplicator)
- Дедупликация по email (primary key)
- Дедупликация по телефону
- Дедупликация по названию организации + адресу
- Слияние данных из разных источников в одну запись

### 5. Квалификатор (Qualifier)
Предварительная оценка лида перед экспортом:
- Наличие контактных данных (email и/или телефон)
- Тип объекта соответствует целевому сегменту
- Объект построен и заселён (не стройка)
- Наличие признаков управления (УК/ТСН)

### 6. Экспортёры (Exporters)

**Bitrix24 Exporter:**
- Формат: CSV для ручного импорта или REST API для автоматического
- Маппинг полей на структуру сделок Bitrix24
- Привязка к воронке "Облако" (Cloud SaaS funnel)

**AI Sales Factory Exporter:**
- Формат: JSON, отправка через webhook `POST /webhooks/lead-form`
- Поля: `company_name`, `contact_name`, `email`, `phone`, `source`, `metadata`
- Metadata содержит: тип объекта, размер, адрес, ИНН, наличие СКУД — для обогащения CHAMP-скоринга

## Потоки данных

### Поток сбора и экспорта

```
1. Запуск парсинга (по расписанию или вручную)
   │
2. Коллекторы собирают сырые данные из N источников
   │
3. Нормализатор приводит к единой модели
   │
4. Обогатитель дополняет данные (ЕГРЮЛ, сайты, соцсети)
   │
5. Дедупликатор удаляет дубли и сливает записи
   │
6. Квалификатор фильтрует нецелевые записи
   │
7. Экспорт:
   ├─ Bitrix24: CSV-файл → импорт в CRM (или REST API → автосоздание сделок)
   └─ AI Sales: JSON → POST /webhooks/lead-form → Lead Scorer → маршрутизация
```

### Внутренняя модель данных (ParsedContact)

```
ParsedContact:
  # Идентификация объекта
  object_name: str          # "КП Сосновый бор"
  object_type: enum         # kp | zhk | bc | warehouse | industrial
  object_address: str       # Полный адрес
  object_region: str        # Регион/область
  object_size: int | null   # Кол-во домов/квартир
  has_security: bool | null # Есть ли охрана/КПП
  has_skud: bool | null     # Есть ли существующая СКУД

  # ЛПР
  contact_name: str | null  # ФИО
  contact_role: str | null  # "Председатель ТСН", "Директор УК"
  contact_email: str | null
  contact_phone: str | null

  # Организация (УК/ТСН)
  org_name: str | null      # "ТСН Сосновый бор"
  org_inn: str | null
  org_ogrn: str | null

  # Мета
  sources: list[str]        # Откуда собраны данные
  collected_at: datetime
  quality_score: float      # 0-1, полнота данных
```

## Технологии и зависимости

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| Язык | Python 3.12 | Совместимость с pass24-ai-sales, богатая экосистема парсинга |
| HTTP-клиент | httpx (async) | Асинхронные запросы, поддержка HTTP/2 |
| Парсинг HTML | BeautifulSoup4 / lxml | Извлечение данных с сайтов |
| Работа с данными | pandas | Нормализация, дедупликация, экспорт CSV |
| Хранилище | SQLite (локальное) или PostgreSQL | Хранение собранных контактов между запусками |
| Очередь задач | Опционально: Celery / asyncio | Параллельный сбор из нескольких источников |
| Логирование | По правилам `agent_docs/guides/logging.md` | Markdown-логи в `logs/` |

## Нефункциональные требования и ограничения

- **Rate limiting:** Соблюдать лимиты API источников (2GIS, Яндекс), задержки между запросами.
- **Робастность:** Парсинг не должен падать целиком из-за ошибки одного источника.
- **Идемпотентность:** Повторный запуск не создаёт дубликатов.
- **Юридические ограничения:** Собирать только данные из открытых источников. Не парсить персональные данные физических лиц (только контакты организаций и должностных лиц).
- **Качество данных:** Экспортировать только записи с quality_score > порогового значения.
