# Интеграция PASS24 Parser ↔ AI Sales Factory

Сценарий взаимодействия парсера с мульти-агентной системой продаж.

---

## Общая схема

```
ПАРСЕР (pass24-parcer-all)              AI SALES FACTORY (pass24-ai-sales)       BITRIX24
─────────────────────                   ──────────────────                       ────────

[1] DDG → сайты КП
[2] Скрапинг контактов
[3] ЕГРЮЛ → ФИО председателя
[4] Дедупликация
[5] Квалификация (quality ≥ 0.4)
         │
         ├─── CSV ──────────────────────────────────────────────────────────────→ Импорт сделок
         │                                                                       (ручная проверка)
         │
         └─── POST webhook ───────────→ [6] Lead Generator
                                             создаёт лид в PostgreSQL
                                             дедуплицирует по email
                                                   │
                                        [7] Lead Scorer
                                             CHAMP-скоринг через LLM + RAG
                                             metadata парсера → точный скоринг
                                                   │
                                        ┌──────────┼──────────┐
                                        │          │          │
                                     ≥70 hot    40-69 warm   <40 cold
                                        │          │          │
                                  [8] Outreach  [9] Nurturing  [10] Long-term
                                   персональное  3 письма:      4 письма:
                                   КП за минуты  день 1/4/10    день 1/7/21/45
                                        │          │          │
                                        └──────────┼──────────┘
                                                   │
                                        [11] Bitrix24 sync ────────────────────→ Сделка в воронке
                                             двусторонняя                        "Облако"
                                             синхронизация
```

## Формат webhook

Парсер отправляет `POST /webhooks/lead-form` в AI Sales Factory.

```json
{
  "source": "parser",
  "form_id": "pass24-parser-import",
  "contact_name": "Иванов Иван Иванович",
  "phone": "+79001234567",
  "email": "ivanov@kp-president.ru",
  "company_name": "ТСН КП Президент",
  "metadata": {
    "object_name": "КП Президент",
    "object_type": "kp",
    "object_address": "МО, Калужское шоссе, 24 км",
    "object_size": 120,
    "has_security": true,
    "has_skud": false,
    "management_type": "tsn",
    "org_inn": "5032123456",
    "org_ogrn": "1125032001234",
    "contact_role": "Председатель",
    "quality_score": 0.85,
    "pre_champ_score": 65
  }
}
```

Реализация: `src/pass24_parser/exporters/ai_sales.py`.
URL задаётся в `.env` → `AI_SALES_WEBHOOK_URL`.

## Как metadata усиливает CHAMP-скоринг

Без парсера AI Sales получает голый лид (имя, телефон). С парсером Lead Scorer использует metadata для точного скоринга:

| Поле metadata | CHAMP-dimension | Логика Lead Scorer |
|---|---|---|
| `has_security: true` | Challenges + | Есть бюджет на безопасность, знают проблему |
| `has_skud: false` | Technical Readiness + | Нет конкурирующей системы |
| `object_size: 120` | Money + | Большой объект = достаточный бюджет |
| `management_type: tsn` | Authority + | 1 ЛПР, не нужно собрание собственников |
| `contact_role: Председатель` | Authority + | Подтверждение полномочий |
| `pre_champ_score: 65` | Все | Предварительная оценка от парсера |

**Пример**: Lead Scorer получает metadata, даёт score 75 → лид уходит в Outreach Agent → через минуты председатель ТСН получает персональное КП с учётом размера посёлка и наличия охраны.

---

## Маршрутизация лидов в AI Sales

| CHAMP Score | Категория | Агент | Действие |
|---|---|---|---|
| ≥ 70 | Hot | Outreach Agent | Персональное КП за минуты |
| 40–69 | Warm | Nurturing Agent | 3 письма: день 1 / 4 / 10 |
| < 40 | Cold | Nurturing Agent | 4 письма: день 1 / 7 / 21 / 45 |

## Формат CSV для Bitrix24

Параллельно с webhook парсер экспортирует CSV для ручного импорта.

```csv
Название сделки,Контакт,Должность,Телефон,Email,Компания,ИНН,Источник,Комментарий
"КП Президент — подключение PASS24","Иванов И.И.","Председатель","+79001234567","ivanov@kp-president.ru","ТСН КП Президент","5032123456","parser","KP, 120 домов, МО, есть охрана, нет СКУД, pre-CHAMP: 65, quality: 0.85"
```

### Маппинг полей

| ParsedContact | Bitrix24 | Примечание |
|---|---|---|
| `object_name` | Название сделки | Формат: `{name} — подключение PASS24` |
| `contact_name` | Контакт → ФИО | Связанный контакт |
| `contact_phone` | Контакт → Телефон | |
| `contact_email` | Контакт → Email | |
| `org_name` | Компания → Название | Связанная компания |
| `org_inn` | Компания → ИНН | |
| `contact_role` | Контакт → Должность | |
| — | Источник | Всегда `"parser"` |
| Всё остальное | Комментарий | Размер, СКУД, охрана, scores |

### Привязка к воронке

- Воронка: **Облако (Cloud SaaS)**
- Стадия: **Новая**
- Ответственный: по региону или round-robin

Реализация: `src/pass24_parser/exporters/bitrix24.py`.

## Фазы запуска

### Фаза 1: Валидация (1–2 недели)

```
Парсер → CSV → ручной импорт в Bitrix24 → менеджеры обзванивают ТОП-50
```

- Источники: DDG + скрапинг сайтов КП МО
- Обогащение: ЕГРЮЛ
- Экспорт: только CSV
- **Критерий перехода**: >70% контактов актуальны (телефон/email работают)

### Фаза 2: Масштабирование (2–4 недели)

```
Парсер → CSV + webhook → AI Sales автоматически обрабатывает
```

- Добавить: Яндекс Карты + больше каталогов КП
- Подключить: `AI_SALES_WEBHOOK_URL`
- Параллельный экспорт: CSV в Bitrix24 + webhook в AI Sales
- **Критерий перехода**: AI-обработка даёт конверсию ≥ ручной

### Фаза 3: Автоматизация (ongoing)

```
Парсер по расписанию → webhook → AI Sales = основной канал
```

- Основной поток: AI Sales Factory
- CSV: только для аудита и отчётности
- Расписание: еженедельно по новым регионам
- Расширение: ЖК и БЦ
- Инкрементальный парсинг: только новые объекты

## Метрики для отслеживания

| Метрика | Источник | Цель |
|---|---|---|
| Конверсия parsed → qualified (CHAMP ≥ 40) | AI Sales | > 60% |
| Конверсия parsed → hot (CHAMP ≥ 70) | AI Sales | > 20% |
| Конверсия parsed → won | Bitrix24 | > 5% |
| Среднее quality_score по батчу | Парсер | > 0.6 |
| % дубликатов с существующей базой | Bitrix24 | < 10% |

## Обратная связь

```
1. Экспорт батча → AI Sales / Bitrix24
2. Через 2–4 недели: выгрузка результатов (won / lost / no response)
3. Анализ: какие источники дают лучшую конверсию?
4. Корректировка: фильтры, приоритеты источников, пороги quality_score
5. Повторный парсинг с улучшенными параметрами
```
